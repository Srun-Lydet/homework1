Safe HTML & JavaScript Injection Awareness Lab

For local classroom learning only — learn how unsafe rendering happens and how to prevent it.

Teacher note: This lesson is defensive. Do not test injection on real websites or systems you do not own. This file uses harmless examples to teach why user input must be validated and escaped.

1. Main Concept

Injection happens when an application treats user input as code or HTML instead of normal text. In frontend JavaScript, the common dangerous sink is innerHTML.

Unsafe: innerHTML Safer: textContent Avoid: eval() Validate input Escape output

4. Code Comparison

Unsafe pattern

// Do not use with user input
result.innerHTML = userInput

Safe pattern

// Better for displaying user text
result.textContent = userInput

5. JavaScript Risk Patterns to Avoid

Risk Case Dangerous Pattern Safe Practice
HTML rendering element.innerHTML = userInput element.textContent = userInput
Inline event attributes <button onclick="..."> button.addEventListener('click', handler)
Running strings as code eval(userInput) Never run user input as code
Unsafe URLs link.href = userInput without validation Allow only trusted protocols like https://

6. Safe URL Validation Example

function isSafeUrl(url) {
  return url.startsWith('https://') || url.startsWith('http://')
}
